Telegram Publishing

Telegram Bot API Automation Notifications Intermediate

Build Claude-powered tasks that send messages, reports, summaries, and alerts directly to a Telegram channel or group. Claude writes the content; the Telegram Bot API delivers it — all triggered from a single prompt.

What you'll learn

  • How to create a Telegram bot and obtain a Bot Token
  • How the Telegram Bot API sendMessage endpoint works
  • How to ask Claude to compose and publish content via curl or Python
  • How to schedule Claude tasks that push periodic reports to Telegram
  • How to format messages with Markdown and inline buttons

How it works

  • You create a bot with @BotFather on Telegram and get a token
  • You find your chat or channel ID (forward a message to @userinfobot)
  • Claude generates the content and calls the Bot API to send it
  • The API endpoint is a simple HTTPS POST — no library required
  • You can run this on a schedule with a cron job or GitHub Actions

Step 1 – Create your bot

Do this in Telegram

Open Telegram and message @BotFather. Send /newbot, choose a name and username, and copy the token it gives you. Store it as an environment variable: TELEGRAM_TOKEN.

Step 2 – Send a test message

Try this prompt

"Using my TELEGRAM_TOKEN and CHAT_ID environment variables, send a test message to my Telegram channel that says 'Hello from Claude Code! 🤖' using a curl command."

Step 3 – Build a publishing workflow

Try this prompt

"Read all the markdown files in the /reports folder, summarise the key findings into a clean Telegram message (use bold text for headings and bullet points), then send it to my TELEGRAM_CHANNEL_ID. Use Python with the requests library."

Example API call

curl one-liner

"curl -s -X POST https://api.telegram.org/bot$TELEGRAM_TOKEN/sendMessage -d chat_id=$CHAT_ID -d text='Your message here' -d parse_mode=Markdown"

Ideas to build

  • Daily standup bot: Claude reads your git log and posts a summary each morning
  • Error alerting: Claude monitors logs and messages you when something breaks
  • Content publisher: Claude drafts posts and sends them to a channel for review
  • Project status reports: Claude reads open issues and pushes a weekly digest
