Flows

Agentic Workflows Intermediate

Flows are multi-step agentic workflows where Claude reasons about a goal, takes actions (reading files, running commands, editing code), checks the result, and loops until the task is done — all without you having to micromanage each step.

What you'll learn

  • What "agentic" means and how Claude plans and acts in loops
  • How to write goal-oriented prompts that trigger a full flow
  • How Claude uses tools (file read, bash, edit) inside a flow
  • How to review, pause, and guide flows mid-execution
  • When flows are appropriate vs. simple one-shot prompts

How flows work

  • You give Claude a high-level goal rather than step-by-step instructions
  • Claude breaks the goal into a plan and executes actions sequentially
  • After each action, Claude checks the result and decides the next step
  • The loop continues until the goal is met or Claude asks for input
  • You can interrupt at any time to redirect or approve risky steps

How to start with Claude Code

Try this prompt

"Look at my entire codebase, find all TODO comments, group them by file and priority, then create a markdown file called TODO.md summarizing them. Run any commands you need to get the full picture."

Tips for effective flows

  • State the desired end state clearly — not a list of steps, but a goal
  • Tell Claude what it's allowed to do: read files, run tests, edit code
  • Add constraints: "don't delete anything", "ask before pushing"
  • Use a CLAUDE.md file to set persistent rules for every flow
  • Review the todo list Claude generates — it's your flow's roadmap
