YouTube Research Tool

MCP YouTube API NotebookLM Automation Intermediate

Claude Code finds the best YouTube videos on any topic and sends them directly to NotebookLM — creating a queryable research notebook with zero copy-pasting. The key is the notebooklm-mcp server, which gives Claude native tool access to NotebookLM.

How it works end-to-end

  • You type: /youtube-research transformer architecture for beginners
  • Claude searches YouTube for the 10 best videos on your topic
  • Claude calls NotebookLM MCP tools to create a notebook and add each video URL as a source
  • Claude queries the notebook and returns a deep research summary — all in one turn
  • No browser. No copy-paste. No manual steps.

What makes this possible: notebooklm-mcp

notebooklm-mcp is an MCP server that exposes NotebookLM as a set of tools Claude can call directly — create notebooks, add sources, run queries, and get citation-backed answers. Authentication is handled once via a browser login that saves a persistent session.

  • Provides tools: create_notebook, add_source, query_notebook, and more
  • Supports YouTube URLs, web pages, PDFs, and text as sources
  • Returns answers with citations pointing back to specific source moments
  • One-time Google login saves a cookie file — subsequent runs need no browser interaction

Step 1 — Install notebooklm-mcp

Prompt to Claude Code

"Install the notebooklm-mcp package and configure it as an MCP server in my Claude Code settings. I want Claude to be able to create NotebookLM notebooks, add YouTube URLs as sources, and query them — all without leaving the terminal."

Claude will run npm install -g notebooklm-mcp (or the pip equivalent) and add the server config to your ~/.claude/settings.json under the mcpServers key.

{
  "mcpServers": {
    "notebooklm": {
      "command": "notebooklm-mcp",
      "args": []
    }
  }
}

Step 2 — Authenticate once

  • Run notebooklm-mcp auth in your terminal
  • A browser window opens — log in with your Google account
  • Your session is saved to a local cookie file
  • That's it — all future Claude calls to NotebookLM reuse this session automatically

Step 3 — Create the /youtube-research skill

Prompt to Claude Code

"Create a skill called /youtube-research. It should: (1) search YouTube for the 10 best educational videos on the topic in $ARGUMENTS, (2) use the notebooklm MCP tools to create a new notebook named after the topic, (3) add each video URL as a source and wait for ingestion, (4) query the notebook with 'Give me a comprehensive research summary with key insights, points of debate, and practical takeaways', and (5) return the full response. Save it to .claude/commands/youtube-research.md."

The skill file

You are a YouTube research assistant with access to NotebookLM via MCP tools.

Topic to research: $ARGUMENTS

Follow these steps exactly:

## 1. Find videos
Search YouTube for the 10 most relevant, educational, and recent videos on
the topic. Prefer content from authoritative channels. Avoid Shorts (under
3 minutes). Collect the full YouTube watch URL for each video.

## 2. Create a notebook
Use the notebooklm `create_notebook` tool with the name:
"Research: $ARGUMENTS"

## 3. Add sources
For each of the 10 YouTube URLs, call the notebooklm `add_source` tool.
Wait for all sources to finish ingesting before proceeding.

## 4. Query for deep research
Call the notebooklm `query_notebook` tool with this prompt:
"Provide a comprehensive research report on the topic. Include:
- Core concepts and definitions
- Key insights and arguments from across the videos
- Points where speakers agree or disagree
- Practical recommendations and takeaways
- Suggested next steps for deeper learning"

## 5. Return results
Present the notebook's response in full. Include the notebook name and
a list of the source videos at the end so the user can open them directly.

Step 4 — Run it

In Claude Code terminal

/youtube-research how large language models actually work

Claude searches YouTube, creates the NotebookLM notebook, adds all 10 videos, and returns a deep research report — fully automated in a single command.

Going further

  • Reuse notebooks: Ask Claude to query an existing notebook by name — "Search my 'Research: LLMs' notebook for anything about fine-tuning"
  • Multi-topic notebooks: Run the skill with several related topics and add all videos to one notebook for cross-topic synthesis
  • Audio Overview: Use the create_audio_overview MCP tool to generate a podcast-style summary of all sources
  • YouTube Data API: For more precise video discovery, swap Claude's web search for a Python script backed by the YouTube Data API v3 — get free API keys from Google Cloud Console
  • Scheduled research: Use Claude Code's /schedule skill to run /youtube-research weekly and keep a notebook current on a fast-moving topic
