YouTube Research Tool

YouTube API NotebookLM Research Intermediate

Build a Claude skill that searches YouTube for the best videos on any topic, curates the links, and prepares them for NotebookLM — giving you a queryable AI research assistant trained on video content.

What you'll build

  • A /youtube-research skill that takes a topic and returns curated video links
  • A pipeline that feeds those links into NotebookLM as sources
  • A research notebook you can query conversationally for deep insights
  • Optional: a script to automate video discovery with the YouTube Data API

How the pipeline works

  • Step 1 – Discover: Claude searches YouTube for videos matching your topic using web search or the YouTube Data API
  • Step 2 – Curate: Claude filters results by relevance, view count, and recency, and returns a clean list of URLs
  • Step 3 – Import: You paste the URLs into NotebookLM as sources — it ingests the transcripts automatically
  • Step 4 – Research: Ask NotebookLM questions and get answers grounded in the video content

Step 1 — Create the skill

Prompt to Claude Code

"Create a skill called /youtube-research. When I run it with a topic, it should search YouTube for the 10 most relevant and recent videos on that subject. Return a markdown list with each video's title, channel, publish date, a one-sentence summary, and a direct YouTube URL. Prioritise educational content from authoritative channels. Save it to .claude/commands/youtube-research.md."

The skill file

Claude will create a file like this at .claude/commands/youtube-research.md:

You are a YouTube research assistant.

Search YouTube for the 10 most relevant, high-quality, and recent videos
on the following topic: $ARGUMENTS

For each video return:
- **Title** – the full video title
- **Channel** – the channel or creator name
- **Published** – approximate publish date
- **Summary** – one sentence describing what the video covers
- **URL** – the full YouTube watch URL (https://www.youtube.com/watch?v=...)

Focus on educational, authoritative content. Prefer videos with transcripts
enabled (most public videos have auto-generated captions). Avoid shorts
under 3 minutes. Sort by relevance to the topic.

After the list, output a "## NotebookLM Import" section that contains only
the bare URLs, one per line, so the user can copy-paste them directly into
NotebookLM's "Add source → Website URL" dialog.

Step 2 — Run the skill

In Claude Code terminal

/youtube-research transformer architecture explained for beginners

Claude returns a curated list with titles, summaries, and URLs — plus a clean URL block at the bottom ready to paste into NotebookLM.

Step 3 — Import into NotebookLM

  • Open NotebookLM at notebooklm.google.com and create a new notebook
  • Click Add source and choose Website URL
  • Paste each YouTube URL — NotebookLM fetches the video transcript as its source
  • Add all 10 URLs (NotebookLM supports up to 50 sources per notebook)
  • Wait ~30 seconds for ingestion, then start asking questions in the chat panel

Step 4 — Query your research notebook

Once your videos are loaded, try queries like:

  • "Summarise the key concepts explained across all these videos"
  • "What do the speakers disagree on?"
  • "Give me a study guide based on this content"
  • "What practical advice did the creators recommend?"
  • "Which video goes deepest on [specific subtopic]?"

Optional — Use the YouTube Data API

For more precise results, connect Claude to the official YouTube Data API v3:

Prompt to Claude Code

"Create a Python script youtube_search.py that uses the YouTube Data API v3 to search for videos on a given topic. Accept the topic as a CLI argument. Return the top 10 results as JSON with videoId, title, channelTitle, publishedAt, and description. Store the API key in a .env file."

Get a free API key from Google Cloud Console — the YouTube Data API has a generous free tier (10,000 units/day).

Tips for better research

  • Add a date filter to your skill prompt: "published in the last 12 months" for fresh content
  • Run the skill multiple times with different angles on your topic to broaden coverage
  • Include a mix of introductory and advanced videos for well-rounded notebooks
  • NotebookLM's Audio Overview feature can synthesise all sources into a podcast-style summary
  • Use NotebookLM's citation links to jump directly to the part of a video transcript that answers your question
