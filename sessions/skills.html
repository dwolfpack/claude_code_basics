Skills

Slash Commands Automation Beginner

Skills are reusable slash commands you define for Claude Code. Each skill is a markdown file with a system prompt and instructions — invoke it with /skill-name and Claude instantly takes on that role.

What you'll learn

  • How skills (slash commands) work in Claude Code
  • How to create a custom skill with a markdown file
  • How to invoke skills and pass arguments
  • Best practices for writing clear skill prompts

How skills work

  • Skills live in .claude/commands/ as .md files
  • The filename becomes the slash command — review.md → /review
  • The file content is the prompt Claude receives when you invoke the skill
  • Use $ARGUMENTS in the file to pass dynamic input at runtime
  • Skills can be scoped to a project or defined globally for all projects

How to start with Claude Code

Try this prompt

"Create a skill called /review that does a thorough code review of the files I pass to it. It should check for bugs, readability, security issues, and suggest improvements. Save it to .claude/commands/review.md."

Example skill file

.claude/commands/review.md

"You are a senior software engineer performing a code review. Review the following files for bugs, security issues, and readability problems. Be specific and actionable. Files: $ARGUMENTS"
