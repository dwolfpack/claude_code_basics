Subagents

Parallel Tasks Orchestration Advanced

Subagents are specialized Claude instances spawned by an orchestrator to handle a focused piece of work. The orchestrator delegates, the subagent executes, and the result flows back — enabling parallelism and better focus on large tasks.

What you'll learn

  • The difference between an orchestrator and a subagent
  • When to break a task into parallel vs. sequential subagents
  • How subagents are isolated and what context they receive
  • How to prompt Claude to spawn and coordinate subagents
  • How the Claude Agent SDK enables subagent orchestration in code

How subagents work

  • An orchestrator Claude receives a high-level task and a set of tools
  • It identifies subtasks that can run independently and spawns subagents for each
  • Each subagent gets its own context, instructions, and tool access
  • Subagents run in isolation — they cannot see each other's work by default
  • Results are returned to the orchestrator, which combines them into a final output

How to start with Claude Code

Try this prompt

"You are an orchestrator. I have three independent tasks: (1) write unit tests for auth.js, (2) refactor utils.js for readability, (3) update README.md to reflect recent changes. Spawn a subagent for each task and run them in parallel, then summarize what each one did."

Agent SDK example

When to use the Agent SDK

"Build me a Python script using the Claude Agent SDK that orchestrates two subagents: one that searches the codebase for security issues and one that checks for performance bottlenecks. Collect both reports and print a combined summary."
