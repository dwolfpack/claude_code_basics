GitHub Profile

GitHub Portfolio Open Source Beginner

Your GitHub profile is your passive portfolio — it speaks for you whether you're active or not. Recruiters open it, scroll through your pinned repos, glance at your README, and form an impression in under 30 seconds. Make those seconds count.

Profile README — your digital handshake

  • Create a repo named exactly after your GitHub username — this unlocks the special profile README at the top of your page
  • Include: a 2–3 line bio, your tech stack (with badge icons), what you're currently building or learning, and links to LinkedIn, portfolio, and email
  • Add a GitHub Stats card via github-readme-stats to show language usage and contribution activity
  • Keep it skimmable — a few tasteful visuals and short sections, not a wall of text

Pinned repositories — your six most valuable slots

  • Pin 4–6 repos that represent your strongest, most diverse work — many recruiters will not scroll past these
  • Choose projects that show breadth (different stacks or problem domains) and depth (more than a tutorial copy)
  • It is fine to pin an in-progress project — it signals active learning. Just make sure it has commits, a clear README, and visible intent
  • Unpin repos with no README, no description, and no activity — they make the whole profile look abandoned

Project READMEs — the biggest differentiator

  • Every public pinned repo must have a quality README: what it does and why, setup/installation steps, usage examples, and a screenshot or demo GIF
  • Add a table of contents for longer READMEs. Include an assets/ folder for screenshots
  • A well-written README is the single biggest difference between an impressive repo and an ignored one — it shows you care about communication, not just code
  • Use badges (build status, license, version) at the top for a professional look

Contribution activity

  • Consistent activity over time is a positive signal — it shows ongoing learning. But do not obsess over an all-green graph
  • Quality beats quantity: meaningful commit messages, merged PRs, and resolved issues signal far more than raw commit counts
  • Enable "Include private contributions on my profile" in settings — your professional work at a company will appear as activity without revealing code
  • One open-source contribution — even fixing a typo in docs — stands out significantly to anyone who looks

Open source & GitHub Pages

  • Contributing to established open-source projects is one of the highest-signal activities on GitHub — start small: fix a bug, improve docs, add a test
  • Participate in Hacktoberfest (October) for a structured way to get your first contributions
  • Host your portfolio site on GitHub Pages — it creates a live link on your profile and shows you know how to deploy
  • Update your pinned repos and README regularly — a profile that looks stale suggests disengagement

Prompt Claude to write your README

Try this prompt

"Write a professional README for my GitHub project. It's a weather dashboard built with React and the OpenWeather API. Include: a short description, tech stack badges, features list, setup instructions, a placeholder for a screenshot, and an MIT license section. Make it clean and employer-ready."
